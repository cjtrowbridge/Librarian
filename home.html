Librarian

Librarian is a project to create a natural language interface for accessing knowledge from sources like wikipedia.

Why?

Currently, this functionality is available using Google Assistant. There are several problems. Google has become fairly scary and evil, and Assistant is in the Google cloud. You do not have control of it, and it does not work if you don't have a decent connection to Google.

How?

There are many opportunities to improve on the technologies mentioned above which feature similar funtionality. For one, wikipedia features distributed access. One may already freely clone the entire thing onto one's mobile device and peruse it without access to the internet. Using tools like IPFS and BitTorrent, this is a mutually beneficial and interdependent service which many citizens can participate in; sharing this valuable content securely between nodes while accessing that content freely.

While Google Assistant can read abstracts of articles or recite quotes from them, it makes no real attempt to provide deep understanding of the content to users. There is no way to get sucked into a long series of articles and find interesting and serendipitous learning through Google Assistant. A natural language interface for ongoing conversations about knowledge would be not just an improvement, but a quantum leap into a much more valuable and interesting domain which Google is not attempting to enter.

Future Plans;

  • Currently, I am writing initial primitives for accessing and summarizing content from public Wikipedia APIs.
  • Next comes forming a precis based on arbitrary content.
  • Then text to speech option for the precis.
  • Then deeper contextual work can start.
